OVERVIEW
--------
A complete stock tracking system was designed and built, integrated into the
existing PHP MVC volunteer management application. The system covers the full
stock lifecycle: setting up categories and items, recording stocktakes, logging
deliveries (stock in), recording stock usage (stock out), recording damaged or
spoiled stock, and viewing current stock levels with a full calculation breakdown.

Nine new pages were added to the application under a new "Stock" menu group.


--------------------------------------------------------------------------------
DATABASE
--------------------------------------------------------------------------------

Tables used (pre-existing, created by client):
  - stock_category   : id, Name
  - stock            : id, Name, Code, category_id
  - stock_movement   : id, stock_id, movement_type, qty, unit, unit_qty,
                       movement_date

Movement types stored in stock_movement.movement_type:
  - stocktake_adjustment  : sets the absolute stock level at a point in time
  - delivery              : adds stock to inventory
  - stockout              : records stock taken from inventory
  - damaged               : records stock damaged or spoiled
  - reversal              : (reserved for future use)

Stock level calculation (date-based):
  Current Qty = (last stocktake qty)
              + (sum of deliveries with movement_date > last stocktake date)
              - (sum of stockouts with movement_date > last stocktake date)
              - (sum of damaged with movement_date > last stocktake date)

  The cutoff is the movement_date of the most recent stocktake_adjustment
  record. All earlier movements are excluded, regardless of row insertion
  order. Items with no stocktake have an implied baseline of zero.

Schema fixes applied (stock_setup.sql):
  - Added AUTO_INCREMENT to stock_movement.id
  - Added movement_date DATETIME DEFAULT CURRENT_TIMESTAMP to stock_movement

NEW FILES CREATED
--------------------------------------------------------------------------------

Database Table Classes (app/database/table/)
  - StockCategoryTable.php   : CRUD table for stock_category
  - StockTable.php           : CRUD table for stock; includes
                               getstockwithlevels() — complex correlated
                               subquery calculating current qty per item
  - StockMovementTable.php   : CRUD table for stock_movement; includes
                               getmovementsbytype($type) JOIN query and
                               insertstocktake() helper

Manager Classes (app/controller/manager/)
  - StockCategoryManager.php     : Standard CRUD manager for categories
  - StockManager.php             : CRUD manager; overrides getparents() to
                                   load categories for the dropdown
  - StocktakeManager.php         : Overrides getallrecords() to use
                                   getstockwithlevels(); implements
                                   performaction('stocktake') to bulk-insert
                                   stocktake_adjustment records from form data
  - DeliveryManager.php          : Overrides getallrecords() to load delivery
                                   movements; overrides getparents() to load
                                   stock items; setdefaults() sets
                                   movement_type='delivery' and
                                   movement_date=now()
  - StockoutManager.php          : Identical to DeliveryManager but for
                                   movement_type='stockout'
  - StockLevelReportManager.php  : Read-only manager; calls
                                   getstockwithlevels() for the report page
  - DamagedStockManager.php      : Same as StockoutManager but records
                                   movement_type='damaged'
  - StockUsageReportManager.php  : Date-range report manager; accepts from/to
                                   dates via setdaterange(); calls
                                   getusagereport() on StockMovementTable;
                                   returns empty dataset when no dates set

Form Classes (app/view/form/)
  - StockCategoryForm.php     : Standard CRUD form; blue banner header;
                                Name field only
  - StockForm.php             : Standard CRUD form; teal banner header;
                                Name, Code, and Category dropdown fields
  - StocktakeForm.php         : Custom form (no standard CRUD buttons);
                                purple banner header; renders a category-
                                grouped table with qty and unit inputs per
                                item; "Save Stocktake" submit button
  - DeliveryForm.php          : CRUD form (new/save/cancel/reset only —
                                no edit or delete of historical records);
                                green banner header with upward arrow;
                                stock item dropdown, qty, unit, unit_qty
  - StockoutForm.php          : Same as DeliveryForm; amber banner header
                                with downward arrow
  - StockLevelReportForm.php  : Read-only report; green banner header;
                                category-grouped 8-column table: Item, Code,
                                Last Stocktake date, Stocktake Qty,
                                + Deliveries, − Used, − Damaged, = Current;
                                current qty shown green (positive) or red
                                (zero/negative); no action buttons;
                                "Export CSV" button downloads all 9 columns
                                as stock-levels-yyyymmdd-hhmm.csv
  - DamagedStockForm.php      : Same as StockoutForm; dark red/burgundy
                                banner header with warning triangle icon;
                                hidden field value='damaged'
  - StockUsageReportForm.php  : Date-range report; blue banner header;
                                From/To date pickers with Generate button;
                                results table grouped by category showing
                                Item, Code, Total Used; summary line with
                                item count; Export CSV button downloads
                                stock-usage-yyyymmdd-hhmm.csv with columns
                                Category, Item, Code, Total Used, From, To;
                                no results shown on first load

BUGS FIXED DURING DEVELOPMENT
--------------------------------------------------------------------------------

1. ClassFactory 21-argument limit
   FormCollection grew to 26 constructor arguments, exceeding the hardcoded
   limit in vendor/fw/factory/ClassFactory.php. Fixed by switching to the
   PHP spread operator (...$pa).

2. PHP 8.2 dynamic property deprecation warnings
   All new manager classes were missing "protected $db;" declarations.
   All new form classes were missing "protected $names = [];" and
   "protected $parents = [];" declarations. Added explicit declarations
   to all 10 new classes.

3. MariaDB trigger definer error
   Triggers and the LogDML stored procedure had a definer of
   sarum964_root@144.6.77.193 (from the original server import). UPDATE
   operations on the user table failed. Fixed by recreating the user table
   triggers and LogDML procedure with definer root@localhost.

5. Usage report To date not inclusive
   The date range query used datetime string comparisons
   (movement_date <= 'yyyy-mm-dd 23:59:59') which could exclude records
   on the To date depending on their stored time. Fixed by using
   DATE(sm.movement_date) <= 'yyyy-mm-dd' so the time component is
   stripped before comparison and all records on the To date are included.

6. Seed script hardcoded stock IDs
   tests/seed_movements.php used hardcoded stock item IDs (15, 16, 17)
   which became stale after the tables were rebuilt. Fixed by looking up
   IDs dynamically by Name at runtime.

7. CSV export heading #NAME? error in Excel
   Column headings starting with +, -, or = were interpreted as formulas
   by Excel, showing #NAME? instead of the heading text. Fixed by renaming
   headers to: Deliveries In, Used, Damaged, Current Qty.

4. menuitem menu_number column
   The Stock Level Report menu item (10_6) was inserted with menu_number=10
   instead of 0, causing it not to appear in the menu. Fixed by updating
   the record and correcting the stock_setup.sql script.
